# 🔄 Churn Intelligence Engine
### AI-Powered Customer Retention Analytics | E-commerce

## 📌 Project Overview
An end-to-end automated churn prediction pipeline built on 
100,000 real e-commerce transactions. Combines machine learning, 
AI-generated insights, and automated data refresh to help 
retention teams act before customers leave.

**AUC-ROC: 0.8356 | 93,350 customers scored | Weekly automation**

---

## 🏗️ Architecture
Raw Data (Olist) → SQL Feature Engineering → XGBoost Model 
→ AI Brief Generation (Groq LLaMA) → Power BI Dashboard 
→ Auto-refresh weekly via scheduler

---

## 📊 Key Findings
- **80.6% churn rate** driven primarily by late deliveries and poor review scores
- **Late delivery rate: 45.88%** — strongest predictor of churn after spend
- **Health & Beauty** is top revenue category ($1.25M) with lowest churn
- **Electronics** shows highest churn risk despite moderate revenue
- AI briefs generated for 20,000+ critical-risk customers automatically

## 🛠️ Tech Stack
| Layer | Technology |
|---|---|
| Data Storage | SQLite |
| Feature Engineering | Python, Pandas, SQL |
| ML Model | XGBoost (AUC 0.8356) |
| AI Layer | Groq API (LLaMA 3.1) |

## 📁 Project Structure
pipeline/
├── load_data.py        # Loads 9 CSVs into SQLite
├── build_features.py   # SQL feature engineering (16 features)
├── train_model.py      # XGBoost training + evaluation
├── ai_layer.py         # AI brief generation per customer
└── export_for_powerbi.py # Builds 4 dashboard datasets

scheduler/
└── run_pipeline.py     # Weekly automation runner

---

## 🔑 Key Features
- **Zero manual intervention** — full pipeline runs on schedule
- **Visible AI reasoning** — every high-risk customer gets a 
  plain-English retention brief generated by LLaMA 3.1
- **Business-ready output** — Power BI dashboard with 4 pages 
  built for different stakeholders
- **Data leakage prevention** — recency excluded from features, 
  used only for label definition

---

## 📈 Dashboard Pages
